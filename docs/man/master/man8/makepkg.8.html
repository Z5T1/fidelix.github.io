MAKEPKG(8) Linux System Administrator's Manual MAKEPKG(8)

NAME

makepkg - creates an installable package

SYNOPSIS

makepkg [-h] [-d ROOT_DIR] [-m METADATA_DIR] -o <OUTPUT_FILE>

DESCRIPTION

Make an installable package from the contents of a directory. The resulting package can be installed with installpkg(8)

Options:

-o <output file>
The name of the file to write the package archive to. By convention, this is of the form name-version-architecture-osrelease-build.tgz Valid file extensions are .tgz and .tbz.
-d <root directory>
The root directory to make the package from. If this is omitted, defaults to the current working directory.
-m <metadata directory>
The directory containing the package metadata. If this is omitted, defaults to ROOT_DIR/var/pkgdb/output_file_basename.
-h
Displays a help message.

METADATA

The following files are permitted in the metadata directory. If using -m to specify a different metadata directory, only these files will be copied and all other files will be ignored.
description
Contains a description of the package (optionally multiline).
README.install
A README to display when installing the package.
README.upgrade
A README to display when upgrading the package.
README.uninstall
A README to display when uninstalling the package.
doinst.sh
A shell script to run after installing the package.
douninst.sh
A shell script to run before uninstalling the package.
